Customer Management API Documentation

This API allows for managing customer records, including creating, retrieving, updating, and deleting customer information.

API Endpoints

Retrieve a Customer (GET)

Endpoint: /customers/{id}

Method: GET

Description: Retrieves details of a specific customer by ID.

Response: JSON object containing customer details.

Example Request:

curl -X GET http://localhost:3000/customers/1

Retrieve All Customers (GET)

Endpoint: /customers

Method: GET

Description: Retrieves a list of all customers.

Response: JSON array of all customer objects.

Example Request:

curl -X GET http://localhost:3000/customers

Add a New Customer (POST)

Endpoint: /customers

Method: POST

Description: Adds a new customer record.

Request Body: JSON object containing the new customer's details.

Example Request:

curl -X POST http://localhost:3001/customers -H 'Content-Type: application/json' -d \
'{
    "id": 4,
    "name": "Jane Doe",
    "role": "User",
    "email": "jane.doe@example.com",
    "phone": "123-456-7890",
    "contacted": false
}'

Update an Existing Customer (PUT)

Endpoint: /customers/{id}

Method: PUT

Description: Updates an existing customer's details.

Request Body: JSON object containing the customer's updated details. ID in body must match id of endpoint url. All fields must be supplied.

Response: JSON array of all customer objects.

Example Request:

curl -X PUT http://localhost:3000/customers/4 -H 'Content-Type: application/json' -d \d
'{
    "id": 4,
    "name": "Jane Doe Updated",
    "role": "Admin",
    "email": "jane.doe.updated@example.com",
    "phone": "987-654-3210",
    "contacted": true
}'

Delete a Customer (DELETE)

Endpoint: /customers/{id}

Method: DELETE

Description: Deletes a specific customer by ID.

Response: JSON object of the remaining customers after deletion.

Example Request:

curl -X DELETE http://localhost:3000/customers/1

Error Handling

API uses standard HTTP status codes to indicate the success or failure of requests.

  • 200 OK: The request was successful.
  • 400 Bad Request: The request was malformed or invalid. (ID in endpoint does not match ID in body for example.)
  • 404 Not Found: The specified resource was not found. (ID Not Found, for example)
  • 500 Internal Server Error: An error occurred on the server side.
